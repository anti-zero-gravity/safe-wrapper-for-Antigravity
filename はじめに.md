# はじめに

このフォルダは、Antigravity の自動Accept機能使用時に危険なコマンドを確認ダイアログで保護（ラップ）するための設定ファイル群です。

# できること

コマンド保護の **設定・解除・確認** だけのシンプル設計です。

| コマンド | 動作 |
|---------|------|
| `/設定` | SafeCmd をインストール |
| `/解除` | SafeCmd をアンインストール |
| `/確認` | 現在の状態を確認 |

設定後は、他のワークスペースでもターミナル（管理者）でも確認ダイアログが有効になります。
このワークスペースを使い続ける必要はありません。

6種類の命令それぞれに対して「自動で許可」か「確認する」かを設定できます。
- 命令項目を移動 → [accept_settings.md](file:///c:/Users/YK-PC/.gemini/antigravity/scratch/safe-wrapper/accept_settings.md) を保存 → `/設定`

## ⚠️ 重要：PowerShellプロファイルについて

`/設定` を実行すると、**PowerShell プロファイル（`$PROFILE`）が上書き**されます。

### 📦 自動バックアップ機能
- **既存プロファイルがある場合**: `Microsoft.PowerShell_profile.ps1$` に自動バックアップ
- **既存プロファイルがない場合**: プレースホルダーファイルを作成（「元々なかった」と記録）
- **既にバックアップがある場合**: 上書きしません（安全）

### � プロファイルマージ機能
既存のプロファイル（Anacondaの `conda init` コードなど）がある場合、**SafeCmdプロファイルの先頭に自動的にコピー**されます。これにより、SafeCmd導入後も既存の環境設定がそのまま維持されます。

### �🗑️ アンインストール時 (`/解除`)
- SafeCmdプロファイルを削除
- **バックアップが本物のプロファイル**: 自動的に復元
- **バックアップがプレースホルダー**: 削除のみ（元々プロファイルがなかったため）

## ⚠️ ラップ適用範囲外

以下の AI ツールは PowerShell を経由しないため、**MessageBox 確認は表示されません**:

### AI ファイル操作ツール（ワークスペース内のみ）
- `write_to_file` - ファイル作成・上書き
- `replace_file_content` - ファイル内容の部分編集
- `multi_replace_file_content` - 複数箇所の編集

### AI が直接できない操作（run_command 経由が必要）
- ファイル削除 → PowerShell `Remove-Item` → ❌ MessageBox 表示
- ファイルリネーム → PowerShell `Rename-Item` → ❌ MessageBox 表示
- ファイル移動 → PowerShell `Move-Item` → ❌ MessageBox 表示
- ワークスペース外の操作 → PowerShell 経由のみ → ❌ MessageBox 表示

## 💡 AI のデフォルト動作

- AI は**デフォルトで PowerShell を使用**
- ユーザーが「cmd で」と明示しない限り、cmd.exe は使われない
- cmd.exe はコマンドオーバーライド不可のため、ラップ設定が効かない
    - ※ `net`, `sc` などの外部コマンドも同様に PowerShell プロファイルの影響を受けないため、阻止できません

**結論:** 通常の操作では自動的にラップが適用される

### ※ UAC（管理者権限）警告について
`Start-Process -Verb RunAs`（管理者権限要求）に対する警告の実装も試みましたが、PowerShell プロファイル内での複雑なパラメータのエスケープ処理が技術的に不安定であり、構文エラーのリスクが高いため、**実装を見送りました**。
現在は Windows 標準の UAC ダイアログに任せる形（デフォルト挙動）となっています。

##  プロファイル適用範囲（`safe-cmd-msgbox.ps1` が代行）

### ✅ 適用されない
- 他のプログラムからの PowerShell 呼び出し
- OS スケジュールタスク
- バッチファイルからの呼び出し
- `-NoProfile` オプション付き実行

### ❌ 適用される
- 対話型 PowerShell（ユーザーが起動）
- AI の `run_command`（デフォルト設定）
